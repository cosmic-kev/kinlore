# Family Photo Tagger — Project Handoff

## What this is
A self-contained family photo organization app built as a single HTML file (`tagger.html`). The goal is to scan old family photos, tag people and dates, and distribute organized photo collections on flash drives to non-technical family members.

## The two-file system
- **`tagger.html`** — the complete tagging tool (open in Chrome or Edge)
- **`index.html`** — the family viewer, generated *by* the tagger, not edited directly

The tagger writes `index.html` and `metadata.json` into the photos folder. Family members just double-click `index.html` — no installation, no internet required, works on Windows in Edge.

## Flash drive folder structure
```
📁 Family Photos/
  ├── index.html        ← generated by tagger, opened by family
  ├── metadata.json     ← tag data, auto-saved by tagger
  └── 📁 photos/
        ├── thumbs/     ← generated by tagger on first open
        └── *.jpg
```

## How tagger.html works
- Open in Chrome/Edge, click "Open Folder", select a photos folder
- On folder open, a dialog shows photo count, existing tag count, and a checkbox to generate/update thumbnails in-browser (no Python needed)
- Thumbnail generation uses Web Workers + OffscreenCanvas for parallel processing (up to 8 threads, auto-scales to `navigator.hardwareConcurrency`)
- Click any photo → full-size image loads in the left pane using `img.decode()` for async decoding, with background preloading of the next 2 photos
- Right panel: tag people (typeahead from existing names, free-type new ones), set a partial date (year / month / day all optional), add notes/caption
- Three action buttons: "Save & Next", "Save", "Discard"
- X button on the image returns to grid view, prompts to save if there are unsaved changes
- `metadata.json` is auto-saved to disk on every Save action
- "Generate Viewer" button bakes all metadata into a self-contained `index.html` for distribution

## metadata.json format
```json
{
  "IMG_001.jpg": {
    "people": ["Grandma", "Dad"],
    "date": "1985-06",
    "caption": "Summer holiday at the lake"
  }
}
```
Dates are partial: `"1985"`, `"1985-06"`, or `"1985-06-15"` — only as specific as known.

## How index.html (the viewer) works
- Fully self-contained — metadata is baked in as a JS variable at generation time, so it works from a flash drive via `file://` with no server needed
- Warm vintage aesthetic: dark ink tones, Playfair Display + Lora fonts, polaroid-style photo cards
- Filter by person (pill buttons), search captions/names, photo count display
- Click any photo → lightbox with full-size image, people, date, caption, prev/next navigation, keyboard arrows

## Key technical decisions
- **No backend, no build tools** — everything is vanilla HTML/CSS/JS in one file
- **File System Access API** (`showDirectoryPicker`) for folder access — requires Chrome or Edge, not Firefox/Safari
- **Viewer metadata injection** — the viewer template is stored as a JSON string inside the tagger; on generation, `__METADATA_JSON__` and `__TITLE__` placeholders are replaced, and escaped template literal syntax (`\`` and `\${`) is unescaped before writing to disk
- **Preload cache** — `Map` of `url → Promise<HTMLImageElement>` prevents decoding the same image twice
- **Dirty tracking** — `isDirty` flag set by any tag/date/caption change, cleared on save or discard

## Things that work well and shouldn't be broken
- The viewer generation unescape step (`.replace(/\\`/g, '\`')`) — critical, without it the viewer's JS crashes silently and no photos render
- The `</script>` escaping inside the viewer template JSON string — must stay as `<\/script>`
- Thumbnail skip logic: compares `lastModified` timestamps, only regenerates if source is newer than existing thumb
- The `fullMeta` loop in `generateViewer` that includes ALL photos (not just tagged ones) in the viewer

## Aesthetic
Warm vintage / nostalgic throughout. CSS variables: `--ink`, `--paper`, `--sepia`, `--accent`. Fonts: Playfair Display (headings), Lora (body). Loaded from Google Fonts. The viewer has a dark background; the tagger has a warm paper background.
